## 二、设计决策摘要

### 2.1 核心架构决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 系统架构 | **模块化单体**，非微服务 | 业务对象复杂、事务关系强，过早拆服务增加复杂度 |
| 前端框架 | React + TypeScript + AG Grid Enterprise | 重表格场景，需要成熟grid内核 |
| 后端框架 | Java 21 + Spring Boot 3 + Flowable | 流程引擎生态成熟，企业级权限强 |
| 数据库 | PostgreSQL 16+ | 强事务、JSON支持、递归查询、未来可接向量搜索 |
| 缓存 | Redis | 登录态、查询缓存、分布式锁 |
| 对象存储 | MinIO(私有) / S3(云) | 报表文件、导入文件 |
| 流程引擎 | Flowable + 自研表单配置层 | 流程走BPMN，表单自己控 |
| ORM策略 | MyBatis(核心) + JPA(配置) | 核心事务模块优先可控SQL |
| 前端表格 | AG Grid Enterprise | 树表格、虚拟滚动、批量编辑 |
| Worker | Java同语言栈独立进程 | 报表导出、批量重算、AI推荐 |

### 2.2 业务模型决策

| 决策 | 说明 |
|------|------|
| 9阶段模型 | 投资估算→目标成本→招标清单→招标控制价→投标报价→合同清单→施工过程→竣工结算→项目复盘 |
| 版本链机制 | "版本头+明细行"模式，支持引用、分叉、锁定、追溯 |
| 合同锁定 | 锁定后不可直接修改，所有调整通过过程单据体现 |
| AI隔离 | AI推荐结果与正式业务数据隔离，不直接覆盖 |
| 专业×单体矩阵 | 项目按专业方向×工程单体组织，支持跨阶段分工 |
| 源系统兼容 | 预留`source_*`字段，支持新点江苏版数据导入 |

### 2.3 V1范围边界

- **纳入V1**: AI清单推荐、AI定额匹配、AI造价预警、AI辅助审核、造价指标积累
- **后续版本**: 智能造价问答、AI材料调差、市场价格智能采集
- **V1不做**: 微服务拆分、Elasticsearch、Kafka、独立规则引擎、通用低代码平台
