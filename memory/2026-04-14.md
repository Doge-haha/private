## D013 — Per-agent MCP Servers 插件完成

**插件路径**: `~/.openclaw/extensions/agent-mcp-servers/index.ts` + `~/.openclaw/workspace/plugins/agent-mcp-servers/`

- 最终方案：**始终预加载**到 `mcporter.json`（不是按需注册）

- `mcporter list` 显示 `glm-search` + `dev__filesystem`
- E2E 验证通过：subagent 内 `mcporter call dev__filesystem.list_directory` 成功返回 `[FILE] test.txt`
- AGENTS.md 委派模板已更新，包含 MCP servers 信息注入说明

**Owner**: 需要在 spawn 时主脑在 task 参数里注入 `MCP Servers` 字段。

## Claude Code Skills 系统深度逆向分析（三轮深挖）

**源码路径**: `/Users/huahaha/WorkSpace/something/src/claude-code-leaked-main/source code/`

### 核心文件清单
| 文件 | 职责 |
|------|------|
| `skills/loadSkillsDir.ts` | Skill 加载主逻辑（managed/user/project/additional/legacy） |
| `skills/bundledSkills.ts` | 内建 skill 注册（verify/remember/loop） |
| `skills/mcpSkillBuilders.ts` | MCP skill 构建器注册（避免循环依赖的间接层） |
| `tools/SkillTool/SkillTool.ts` | SkillTool 实现（inline/fork/remote 三种执行模式） |
| `tools/SkillTool/prompt.ts` | Skill prompt 生成 + token 预算控制 |
| `utils/attachments.ts` | Skill listing 注入 system prompt + 动态发现触发 |
| `utils/skills/skillChangeDetector.ts` | 文件变化监听（chokidar）→ skill 缓存失效 |
| `commands.ts` | Command 注册中心（getCommands/getSkillToolCommands） |

### 架构要点
1. **4层来源优先级**: managed → user → project → additional → legacy commands → plugin → bundled → MCP
2. **三种执行模式**: inline（展开到当前对话）、fork（独立子agent）、remote（AKI/GCS远程加载，实验性）
3. **条件激活**: `paths` frontmatter → 只在操作匹配路径文件时才激活skill
4. **动态发现**: FileRead/Edit/Write 操作后向上遍历找 `.claude/skills/` 目录
5. **Token预算**: 占上下文窗口1%，bundled skills永不截断，其他按预算均分
6. **去重**: realpath级别（文件去重）+ 名称级别（深路径优先覆盖）
7. **权限**: deny rules → remote auto-allow → allow rules → safe-properties auto-allow → ask user
8. **安全**: gitignored目录跳过、MCP skills不执行shell命令

### 对 OpenClaw 的关键启示
- **条件skill（paths frontmatter）** 是最精巧的设计 — 按需激活而非全局加载
- **动态发现机制** — 文件操作触发向上遍历寻找嵌套skill目录
- OpenClaw 的 `<available_skills>` 注入类似但缺少 budget 控制和条件激活
- fork 模式 ≈ OpenClaw 的 sessions_spawn，inline ≈ 直接注入 prompt
