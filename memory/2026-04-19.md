## 030-tender-bill.html 招标清单页面（今日工作重点）

### 文件路径
`projects/xindian-saas-frontend/stages/030-tender-bill.html`

### 业务层次结构修正（两次迭代）

**第一次修正**：从"按专业分单位工程"改为"按楼栋分单位工程"
- 原来错误：单位工程 = 土建 / 安装 / 装修（每个专业一个单位工程）
- 正确结构：建设项目 → 标段 → 楼栋（即单位工程）→ 专业（土建/装修/安装）→ 分部 → 分项 → 清单
- 核心认知：一个楼栋内同时包含土建+装修+安装所有专业

**第二次修正**：措施费/其他费用/税金应该是和土建/装修/安装同一层级
- 这三项是独立费用类别，不属于分部分项
- 应该在每个楼栋下作为独立子节点

### 当前数据规模
- 407条清单（92KB）
- 2个标段（住宅一标段 + 住宅二标段）
- 6个楼栋（1#楼/2#楼/地下室/3#楼/4#楼/）
- 每个楼栋下有5个专业节点：土建 / 装修 / 安装 / 措施费 / 其他费用
