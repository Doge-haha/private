# 新点 SaaS 造价系统 — 前端设计规范 v0.1

> 本目录为 @prime 独立设计工作区，与 Codex 工作区（~/Documents/New project）完全隔离。

## 1. Concept & Vision

面向造价工程师的专业 SaaS 工具，强调**数据密度**与**操作效率**。界面风格参考广联达、晨曦等国内头部造价软件——信息密集、表格优先、少用花哨动画。但在交互细节上借鉴 Linear/Notion 的克制美学：安静但精准。

核心体验原则：
- **表格即一切**：造价软件的灵魂是数据表格，任何页面都以表格为中心
- **少打断多连续**：造价人员每天要录入上百行数据，任何 modal/drawer 打断都是负担
- **状态可见**：版本状态、锁定状态、审核状态一目了然，不需要点进去才知道

### 2.2 字体

- **主字体**：`"PingFang SC", "Microsoft YaHei", "Noto Sans SC", sans-serif`
- **数字/金额**：`"DIN Alternate", "Roboto Mono", monospace`（等宽，保证小数点对齐）
- **字号层级**：
  - 页面标题：18px / 600
  - 模块标题：14px / 600
  - 正文：13px / 400
  - 表格内容：13px / 400（数字用等宽）
  - 辅助文字：12px / 400

### 2.3 间距系统


### 2.4 动效策略

克制，仅用于状态反馈：
- 按钮 hover：`background 150ms ease`
- 下拉展开：`max-height 200ms ease-out`
- 行 hover：`background 100ms ease`
- Toast 通知：`translateY + opacity 250ms`
- **禁止**：页面跳转动画、modal 弹入动画、任何弹性/弹簧效果

## 5. 组件清单

### 5.1 表格组件 (DataGrid)

核心组件，参考 handsontable/suite 风格：
- 虚拟滚动（行数可能上千）
- 列宽可拖拽调整
- 固定列（左侧序号+操作，右侧合计）
- 行多选 + 批量操作
- 单元格编辑（双击 / Enter / Tab 切换）
- 右键上下文菜单
- 展开行（显示子表格，如定额明细）

### 5.2 状态徽章 (StatusBadge)

小标签，圆角，4 种状态颜色组合（见 4.4）。

### 5.3 工具栏 (Toolbar)

固定在表格上方，始终可见。包含新增按钮、操作按钮组、右侧筛选/搜索。

### 5.4 版本选择器 (VersionSelector)

下拉选择器，显示版本名称 + 状态徽章 + 版本号。切换版本时顶部工具栏变色提示（跨版本警告）。

### 5.5 汇总面板 (SummaryPanel)

固定在表格下方，显示：
- 清单项总数
- 系统合价合计
- 综合合价合计
- 费率差（综合/系统，显示百分比）
- 与上一版本的对比（Δ 值）

### 5.6 侧滑抽屉 (SlideDrawer)

用于清单详情、定额明细等二级信息展示。
- 从右侧滑入，宽度 480px（可拖拽调整）
- 关闭：点击遮罩 / ESC / 关闭按钮
- 不打断主表格的光标焦点（编辑器场景友好）

## 6. 技术栈建议

```
框架：      React 18 + TypeScript
UI 库：     Ant Design 5.x（国内企业级首选，表格生态成熟）
表格引擎：  @handsontable/react（行数多时虚拟滚动必备）
状态管理：  Zustand（轻量，够用）
路由：      React Router 6
HTTP：      Axios + React Query（数据获取 + 缓存）
表单：      React Hook Form + Zod（与后端 schema 共用）
构建：      Vite
样式：      CSS Variables + Less（覆盖 Ant Design 主题）
图标：      Lucide React
数字字体：  DIN Alternate（Google Fonts / 本地字库）
```
