## 什么是旁路由？

旁路由是一个放在主路由"旁边"的小网关，所有流量经过它转发到外网。

**Apple TV 做旁路由的优势：**
- 功耗极低：只有 **1W**（普通软路由 5-10W）
- 无需刷机：直接安装 App 即可
- 全家受益：所有设备（手机/电脑/平板）无需装任何 App，走网关代理

---

## 准备工作

### 硬件
- Apple TV 4K 或 Apple TV HD（需 tvOS 17+）
- 已接入家中局域网的主路由器

### 软件（3选1）
| App | 特点 |
|-----|------|
| **Quantumult X** | 功能最全，推荐 |
| **Stash** | 轻量，支持 Clash 规则 |
| **Loon** | 简单易用 |

### 节点/VPN 配置
- 需要已有翻墙节点（SS/V2ray/Trojan/VMess 等）
- 或订阅机场服务

## 教程步骤

### 第一步：在 Apple TV 上安装并配置 Quantumult X

1. 在 Apple TV App Store 搜索 **Quantumult X** 并安装
2. 打开 App，进入「设置」→「服务器」
3. 添加你的节点信息（服务器地址、端口、密码、加密方式）
4. 或者粘贴机场订阅链接 → 自动导入节点
5. 开启 **HTTP 代理**（监听 198.18.0.2:8080 或类似端口）

### 第二步：固定 Apple TV 的 IP 地址

这一步非常关键，旁路由需要稳定的内网 IP。

1. Apple TV → 设置 → Wi-Fi → 已连接的网络 → 划到最底下
2. 配置 IP 地址 → 选「手动」
3. 记录当前 IP，例如：`192.168.1.100`
4. 记住子网掩码、路由器地址（通常是 192.168.1.1）
5. DNS 建议填：`198.18.0.2`（Quantumult X 的本地 DNS 劫持地址）

### 第三步：设置其他设备的网关指向 Apple TV

#### iPhone / iPad
```
设置 → Wi-Fi → 已连接网络 → HTTP代理 → 手动
服务器：填 Apple TV 的 IP（如 192.168.1.100）
端口：填 Quantumult X 的代理端口（如 8080）
```

#### Android
```
设置 → Wi-Fi → 长按已连接网络 → 高级设置 → 代理 → 手动
服务器：填 Apple TV 的 IP
端口：同上
```

#### Windows / macOS
```
系统偏好设置 → Wi-Fi → 高级 → 代理
勾选「Web 代理(HTTP)」和「安全 Web 代理(HTTPS)」
填入 Apple TV 的 IP 和端口
```

#### ⚠️ 注意：每台设备都需要单独设置代理

### 第四步（可选）：用 DNS 劫持实现全局代理

如果不想每台设备都设置代理，可以设置 DNS：

1. 主路由的 DNS 服务器改为 Apple TV 的 IP
2. Apple TV 上的 Quantumult X 开启「增强模式」或「神话模式」
3. 这样所有设备自动走代理，**无需设置 HTTP 代理**

> ⚠️ 部分设备（游戏机、IoT设备）可能需要另外处理

## 常见问题

### Q: 为什么有些设备连上了但不能上网？
**A:** 检查 Apple TV 上的 Quantumult X 是否正常运行，节点是否有效。尝试在 Apple TV 内置浏览器打开 YouTube 测试。

### Q: 间歇性断线？
**A:** 
1. 检查 Apple TV 的 IP 是否变过（建议在路由器里做 IP 绑定/静态 DHCP）
2. 关闭主路由的 IPv6：`主路由设置 → IPv6 → 关闭`
3. DNS 不要用 198.18.0.2 以外的地址

### Q: 网速慢？
**A:** Apple TV 性能有限，跑大流量会吃力。建议看视频够用，打游戏/大文件下载直接用原生翻墙。

## 总结：拓扑图

```
[外网] ←→ [主路由器] ←→ [Apple TV (旁路由)] ←→ [手机/电脑/平板]
                         ↑
                   Quantumult X
                   代理所有流量
```

**核心原理：** Apple TV 作为网关，所有设备 HTTP 代理指向它，它替所有设备完成翻墙。
